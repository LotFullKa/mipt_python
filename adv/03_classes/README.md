# Third Seminar - Тесты для классов Python

Этот проект содержит набор тестов для проверки реализации классов на Python с использованием ООП концепций.

## Структура проекта

```
third_seminar/
├── src/
│   └── __init__.py          # Пакет с реализацией классов
├── tests/
│   ├── test_task01.py       # Тесты для задачи 1: базовый класс Crop
│   ├── test_task02.py       # Тесты для задачи 2: методы __repr__ и __eq__
│   ├── test_task03.py       # Тесты для задачи 3: классовый метод from_dict
│   ├── test_task04.py       # Тесты для задачи 4: property геттеры и сеттеры
│   ├── test_task05.py       # Тесты для задачи 5: Enum и метод is_harvested
│   ├── test_task06.py       # Тесты для задачи 6: классы Cabbage и Rubus
│   ├── test_task07.py       # Тесты для задачи 7: миксин SeasonalMixin
│   ├── test_task08.py       # Тесты для задачи 8: миксин Cruciferous
│   └── test_task09.py       # Тесты для задачи 9: статический метод create_class
└── README.md                # Этот файл
```

## Требования

- **Python**: версия 3.8 или выше
- **pytest**: для запуска тестов

## Установка зависимостей

Установите pytest, если он еще не установлен:

```bash
pip install pytest
```

Или используйте requirements.txt (если есть):

```bash
pip install -r requirements.txt
```

## Запуск тестов

### Запуск всех тестов

Из корневой директории проекта (`third_seminar/`):

```bash
pytest tests/
```

Или из родительской директории:

```bash
pytest third_seminar/tests/
```

### Запуск конкретного тестового файла

Для запуска тестов конкретной задачи:

```bash
pytest tests/test_task01.py
pytest tests/test_task02.py
# и т.д.
```

### Запуск с подробным выводом

Для получения детальной информации о каждом тесте:

```bash
pytest tests/ -v
```

Для еще более подробного вывода (включая print-ы):

```bash
pytest tests/ -vv -s
```

### Запуск конкретного теста

Для запуска конкретного тестового класса или метода:

```bash
# Запуск конкретного класса тестов
pytest tests/test_task01.py::TestCropBasic

# Запуск конкретного теста
pytest tests/test_task01.py::TestCropBasic::test_crop_creation
```

### Показать покрытие кода

Если установлен pytest-cov:

```bash
pip install pytest-cov
pytest tests/ --cov=src --cov-report=html
```

## Описание задач

### Задача 1: Базовый класс Crop
- Создание класса с приватными атрибутами
- Методы get_id, get_seed_date, get_area, get_crop_yield
- Глобальная переменная TOTAL_FARM_AREA

### Задача 2: Магические методы
- Добавление `__repr__` для читаемого представления
- Добавление `__eq__` для сравнения объектов

### Задача 3: Классовый метод
- Метод `from_dict` для создания объекта из словаря

### Задача 4: Property
- Property геттеры и сеттеры для приватных атрибутов
- Валидация данных в сеттерах

### Задача 5: Enum
- Перечисление GrowthStage для стадий роста
- Метод `is_harvested` для проверки готовности к сбору

### Задача 6: Наследование
- Класс Cabbage с методом `head_amount`
- Класс Rubus с приватным полем типа

### Задача 7: Миксины
- SeasonalMixin для учета сезонных особенностей
- Методы `set_season` и `seasonal_bonus`

### Задача 8: Абстрактные классы
- Миксин Cruciferous с абстрактным методом
- Методы для работы с химикатами

### Задача 9: Динамическое создание классов
- Статический метод `create_class` для создания классов во время выполнения

## Примечания

⚠️ **Важно**: Перед запуском тестов необходимо реализовать соответствующие классы в модуле `src/`.

Все классы должны быть импортированы в `src/__init__.py`, чтобы тесты могли их найти:

```python
from src import Crop, Cabbage, Rubus, GrowthStage, SeasonalMixin, Cruciferous, TOTAL_FARM_AREA
```

## Отладка

Если тесты не проходят:

1. Проверьте, что все классы реализованы в `src/`
2. Убедитесь, что классы экспортированы в `src/__init__.py`
3. Запустите тесты с флагом `-v` для подробного вывода
4. Используйте `pytest --pdb` для входа в отладчик при ошибке

## Дополнительные опции pytest

```bash
# Остановиться на первой ошибке
pytest tests/ -x

# Показать локальные переменные при ошибке
pytest tests/ -l

# Запустить только упавшие тесты из прошлого запуска
pytest tests/ --lf

# Показать самые медленные тесты
pytest tests/ --durations=10
```
